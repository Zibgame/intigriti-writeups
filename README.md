# 🛡️ Bug Bounty & Security Research Workspace

## 🔐 Overview

This repository contains my bug bounty research, including real-world vulnerabilities, writeups, tools, and methodologies.

It is designed to:

* Track findings and progress
* Improve exploitation skills
* Build a scalable and professional security workflow

---

## 📂 Repository Structure

### 📁 reports/

Detailed vulnerability writeups:

* Organized by target and vulnerability type
* Each report includes:

  * README.md (full writeup)
  * PoC (video or screenshots)
  * HTTP requests (proof)

---

## ⚡ Focus Areas

* Web application security
* API security
* Broken access control (IDOR)
* Business logic vulnerabilities

---

## 🚀 Methodology

1. Recon & surface mapping
2. Analyze user flows (cart, auth, profile, etc.)
3. Intercept requests (Burp Suite)
4. Modify parameters (IDs, tokens, inputs)
5. Test access control & logic
6. Validate impact across sessions/users

---

## 📈 Goals

* Find high-impact vulnerabilities
* Improve consistency and speed
* Build a strong public portfolio
* Focus on real-world exploitation

---

## ⚠️ Disclaimer

All activities are conducted within authorized bug bounty programs and follow responsible disclosure policies.

No sensitive or unpatched vulnerabilities are publicly disclosed.
